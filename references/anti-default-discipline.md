## §1 克制六律

报告设计的六条铁律。**违反任一条 = 设计缺陷**。

### 律一: 功能色克制

一份报告最多 **4 种情感色** (增长/风险/机会/中性)。

| 允许 | 禁止 |
|:---|:---|
| 绿=增长, 红=风险, 蓝=机会, 灰=中性 | 紫色装饰、粉色高亮、橙色阴影 |
| 同一含义全报告固定一种颜色 | 不同章节用不同颜色表达「增长」 |
| 品牌色只出现在 Hero/Closing/章节号 | 品牌蓝铺满全页做背景 |

### 律二: 动效克制

动效只用于传达层级 (CountUp/Reveal/Progress)，不做装饰性动画。

| 允许 | 禁止 |
|:---|:---|
| CountUp 数字动画 (Hero/KPI 核心指标) | 全页元素同时弹跳入场 |
| Scroll Reveal fade-up (0.5s 缓出) | 无限循环的呼吸灯、脉冲 |
| Hover 微交互 (150ms 阴影升起) | 鼠标轨迹粒子 |
| 进度条渐进填充 | 旋转图标/加载动画做装饰 |

### 律三: 图表克制

**一个 Chapter 最多 2 个图表组件** (紧凑销售报告档放宽至 ≤3, 且子弹阵列/Small Multiples/Sparkline 表格等**密集形态算 1 个组件**——密度靠形态不靠堆图, 见 `chart-selection-guide.md` §7)。删掉不影响叙事的图表。

| 允许 | 禁止 |
|:---|:---|
| 趋势折线 + 归因瀑布 (两个相互配合) | 折线+饼图+柱图+雷达图堆在同一章节 |
| 每张图只承担一种分析任务 | 一张图同时展示趋势、占比和排名 |
| 图表上方有结论性标题 | 图表标题是描述标签 |
| 紧凑档: 一个子弹阵列扫完 8 个战区达成 | 以"紧凑"为名砍掉归因图/达成图 (那是砍分析) |

### 律四: 装饰克制

信息靠排版和留白组织，不靠装饰元素。

| 允许 | 禁止 |
|:---|:---|
| 适度的 1px 分隔线 | 渐变色边框、发光效果 |
| 品牌色横线做 Pull Quote 装饰 | 3D 阴影、浮雕效果 |
| 克制的圆角 (12px) | 不规则图形、斜切背景 |
| 纯色 Badge 标签 | 毛玻璃 Badge、渐变 Badge |

### 律五: 密度克制

**一屏最多 5 个独立信息区块**。超过的折叠或下沉到 Data Detail。

| 场景 | 上限 | 超出处理 |
|:---|:---|:---|
| Hero 首屏 | 1 核心数字 + 1 标题 + 1 副标题 | 不堆 KPI 卡片 |
| KPI Strip | 3-5 项 | 超过 5 项按重要性取舍 |
| Chapter 正文 | 2 组件 (文+图/文+表) | 第 3 个组件移到下一节 |
| Insight Cards | 3-4 张 | 超过 4 张分页或折叠 |

### 律六: 文案克制

Pull Quote 只能是洞察/反转/顿悟。数据复述、空洞鼓励、无方向感慨一律禁止。

| 允许 | 禁止 |
|:---|:---|
| "增速从 22% 降至 12%，不是市场变冷，是去年基数在变高" | "本月签收 8,639 台，同比增长 12.3%" |
| "传统渠道还在惯性走 C，但消费者已经在搜索变频一级能效" | "我们要继续加强努力" |
| "35% 不是终点，而是结构升级的起点" | "总体形势良好，充满信心" |

## §2 Anti-Default 负面清单 (AI 默认审美陷阱)

> 这些是 AI 在没有明确约束时最容易输出的「默认审美」。每一条都需要主动规避。

### 2.1 布局陷阱

| 陷阱 | 表现 | 正确做法 |
|:---|:---|:---|
| **均等 KPI 卡片墙** | 4-6 个大小完全一样的 KPI 卡片排成网格 | KPI Strip 横向一行，或使用 Hero Tile + 小 Tile 的权重差异 |
| **居中 Hero 万金油** | 每份报告都是深色渐变 + 居中大字 | Hero 高度和风格应该匹配报告类型 — 日报不需要 100vh Hero |
| **三等分卡片** | 三列等宽卡片，无论内容多少 | 根据内容量决定列宽，使用 `1fr 2fr` 或 `2fr 1fr` 不对称网格 |
| **全页蓝色渐变** | 整份报告从头到尾都是蓝色系 | 品牌色只出现在 Hero/Closing/章节号，正文区域是白底 |
| **信息堆砌首屏** | 把所有数据塞进第一屏 | 首屏只服务一个第一读者，其余内容向下滚动 |

### 2.2 排版陷阱

| 陷阱 | 表现 | 正确做法 |
|:---|:---|:---|
| **字号无层级** | 标题 20px，正文 16px，差距太小 | 使用 `--text-hero` 到 `--text-xs` 的 7 级跨度制造视觉张力 |
| **全页一种字体** | 标题正文数字全用 Inter 或 DM Sans | 三角色分工: Display + Editorial + Data |
| **等宽间距** | 所有元素间距都是 16px | 章节间 80px 留白，组件间 24px，行内 4px — 用间距表达层级 |
| **无呼吸空间** | Section 之间没有明显的视觉停顿 | 章节间至少 `--space-20` (80px)，加上 `<div class="divider">` 分隔 |

### 2.3 图表陷阱

| 陷阱 | 表现 | 正确做法 |
|:---|:---|:---|
| **默认 ECharts 配色** | 蓝-绿-黄-红 默认色板直出 | 使用 Token 系统的 `--chart-1` 到 `--chart-6` |
| **满屏网格线** | 横竖网格线都显示 | 只保留水平虚线，Y 轴不显示线 |
| **全部带图例** | 每张图表都带一个大图例 | 优先直接标注 (Direct Label)，省掉图例 |
| **默认饼图** | 任何占比分析都用饼图 | 饼图/环形默认禁用; 超过 3 类改用 100%堆叠条形/Treemap |

### 2.4 叙事陷阱

| 陷阱 | 表现 | 正确做法 |
|:---|:---|:---|
| **描述性标题** | "各战区签收情况"、"渠道对比分析" | Action Title: "传统渠道占比下降3.2pp，需加速渠道转型" |
| **数据复述型 Pull Quote** | "本月签收 8,639 台" | 洞察型: "增速从 22% 降至 12%，不是市场变冷，是基数在变高" |
| **空话 CTA** | "加强管理"、"优化策略" | 可执行: "南宁战区 3 日内实地走访，反馈根因" |
| **无方向文案** | "总体表现稳定" | 有方向: "增长动力从规模拉动切换到结构升级" |

## §3 Pre-Build 检查清单 (在写 HTML 之前)

在进入编码之前，对照以下清单确认每一条:

```markdown
## Pre-Build Checklist

- [ ] 报告类型已确认 (strategic_narrative / operational / brief / audit)
- [ ] Governing Thought 已提炼 (一句话，不是描述标签)
- [ ] Visual Contract 已填写 (受众/布局/信息预算/图表政策)
- [ ] Hero 高度已选择 (100vh / 80vh / 60vh / 无 Hero)
- [ ] Chapter 数量已确定 (2-4 个，每个有 PAC 闭环)
- [ ] 核心图表已选型 (每 Chapter ≤ 2 个)
- [ ] 行动项已明确 (每条有对象+动作+期限)
- [ ] Evidence Pack 已确认 (无证据的结论不进主叙事)
```

## §4 生成后检查清单 (交付 HTML 之前)

```markdown
## Post-Build Checklist (Anti-Default)

### 布局
- [ ] 首屏没有均等 KPI 卡片墙
- [ ] Hero 高度匹配报告类型 (非一律 100vh)
- [ ] 章节间有 ≥ 80px 的呼吸空间
- [ ] 一屏内信息区块 ≤ 5 个

### 排版
- [ ] 字体三角色分工正确 (Display/Editorial/Data)
- [ ] 字号有 ≥ 3 级的视觉层级差
- [ ] 数字全部 tabular-nums + font-data
- [ ] 间距全部来自 --space-* token

### 色彩
- [ ] 情感色 ≤ 4 种
- [ ] 同一含义全报告颜色一致
- [ ] 品牌色只出现在 Hero/Closing/章节号
- [ ] 无彩虹色、无渐变 Badge

### 图表
- [ ] 每张图有结论性标题 (非描述标签)
- [ ] 配色使用 --chart-* token
- [ ] 优先直接标注，减少图例
- [ ] 无 3D、无饼图/环形 (≤3 类且明确要求除外)

### 叙事
- [ ] Hero 标题 = Governing Thought
- [ ] Chapter 标题是 Action Title ≤ 30 字
- [ ] Pull Quote 是洞察/反转 (非数据复述)
- [ ] CTA 有对象+动作+期限 (非空话)
```
